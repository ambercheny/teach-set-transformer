Set Transformer Glossary

A compact vocabulary for learning why Set Transformer fits unordered flow cytometry event data and how it produces sample-level predictions.

Data Terms

Terms for the Flow-RBC framing.
Term Meaning Use it when
Set A collection where the elements have no meaningful order. You want to say that shuffling rows should not change the sample-level prediction.
Sequence A collection where order carries information, such as words in a sentence or time points in a time series. You want to contrast Set Transformer with models that treat position as meaningful.
Flow cytometry event One detected particle measurement. In this teaching workspace, after preprocessing, one retained event is treated as one cell row. You want to explain why the model input can be represented as rows of per-cell measurements.
Cell-event row One row in the model input table, containing measured features for a retained event/cell. You want a concrete mental model: one row equals one element in the input set.
Sample The whole blood sample or patient-level example, represented by many event rows. You want to distinguish the model input set from the individual rows inside it.
Sample-level prediction A prediction for the entire set, not for an individual event row. You are describing hematocrit prediction from many RBC measurements.
Event-level representation A learned vector for each event row after the encoder processes the set. You are tracking how the encoder keeps one output representation per input event.
Hematocrit The fraction of blood volume occupied by red blood cells. You are describing the Flow-RBC prediction target from Zhang 2022.

Symmetry Terms

Permutation

A reordering of the elements in a set. For a table of event rows, a permutation means the same rows appear in a different order.

Permutation Invariant

A function is permutation invariant when reordering the input elements does not change the output.

f([a, b, c]) = f([c, a, b])

Permutation Equivariant

A function is permutation equivariant when reordering the input elements reorders the output elements in the same way.

g([a, b, c]) = [A, B, C]
g([c, a, b]) = [C, A, B]

Rule of thumb Invariant means "same answer after shuffling." Equivariant means "same per-element answers, shuffled the same way."

Architecture Terms

Terms for Deep Sets and Set Transformer.
Term Meaning Why it matters
Deep Sets A baseline architecture that encodes each set element, pools across elements, then decodes the pooled vector. It is the simplest reference point for permutation-invariant neural networks.
Pooling An aggregation operation such as sum, mean, or max across event representations. Pooling can make a model invariant, but simple pooling may discard useful interactions between elements.
Attention A weighted lookup mechanism: a query scores keys, then uses those scores to combine values. It lets the model emphasize different event rows depending on the context of the set.
Query The vector asking what information is relevant. Use it when explaining which element or seed is doing the lookup.
Key The vector being compared against a query. Use it when explaining how attention scores are computed.
Value The vector being combined after attention weights are computed. Use it when explaining what information flows to the output.
MAB Multihead Attention Block. The reusable attention block behind Set Transformer components. SAB, ISAB, and PMA are built from MAB.
SAB Set Attention Block. A self-attention block where the set attends to itself. It creates event-level representations that can include interactions with other events.
ISAB Induced Set Attention Block. A scalable attention block that uses learned inducing points. It reduces attention cost from comparing all event pairs to comparing events through a smaller learned bottleneck.
Inducing point A learned vector used by ISAB as an intermediate representation for summarizing set structure. It helps the model scale when the number of events is large.
PMA Pooling by Multihead Attention. A learned pooling operation using seed vectors as queries. It is where Set Transformer turns event-level representations into set-level outputs.
Seed vector A learned query vector used by PMA to ask for a summary of the set. One seed can produce one sample-level output representation; multiple seeds can produce multiple related outputs.

Zhang 2022 Terms

Terms for deep permutation-invariant model stability.
Term Meaning Why it matters
Gradient vanishing Gradients become too small as they flow backward through many layers. Earlier layers may learn slowly or stop learning in deep models.
Gradient exploding Gradients become too large as they flow backward through many layers. Training can become unstable, especially in deeper Set Transformer variants.
Residual connection A connection that adds a block input back to a block output. It helps gradients flow through deep networks.
Clean path A residual path where the input is added back without being transformed first. Zhang 2022 uses this idea to improve deep permutation-invariant architectures.
Layer norm A normalization method that standardizes features within each element representation. It can remove scale information that may matter for real-valued set data.
Set norm A normalization method designed for set inputs, standardizing at the set level while preserving permutation behavior. It is Zhang 2022's alternative to layer norm for permutation-invariant models on set data.
Set Transformer++ A modified Set Transformer using clean-path residual connections and set norm. It is the Zhang 2022 version aimed at training deeper set models more reliably.

Fast Check

Classify each function

  1. mean([x1, x2, x3]): invariant, equivariant, or order-sensitive?
  2. [h(x1), h(x2), h(x3)] with the same h applied to every row: invariant, equivariant, or order-sensitive?
  3. A model that reads event rows one at a time and uses row position as a feature: invariant, equivariant, or order-sensitive?

Answers: invariant; equivariant; order-sensitive.
