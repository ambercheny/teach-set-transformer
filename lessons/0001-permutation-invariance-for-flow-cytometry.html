Lesson 1

Permutation Invariance For Flow Cytometry

Learn the one symmetry that makes set models useful for flow cytometry: reordering cell rows should not change a sample-level prediction.

The Win

By the end of this lesson, you should be able to classify a function as permutation invariant, permutation equivariant, or order-sensitive. You should also be able to explain why hematocrit prediction from many RBC event rows is a set-input problem.

Vocabulary checkpoint Keep the glossary open if the words set, event, sample, invariant, or equivariant feel unstable.

Set, Not Sequence

A sequence has meaningful order. A sentence changes when its words are shuffled. A time series changes when its time points are shuffled. A set has no meaningful order.

Flow Cytometry As Set Data

Flow cytometry data is naturally set-like. At a high level, flow cytometry aligns particles or cells in a single stream, sends light from an optical source such as a laser through or around each passing particle, and records the resulting signal with detectors. Those signals become per-cell measurements that can be used to characterize cells or particles. In flow cytometry literature, sometimes we refer to a cell measurement as an event. For simplicity, we refer to cells.

Conceptual view: cells pass one at a time through the optical path; each retained event becomes one row of measurements.

The order in which cells pass through the instrument has no canonical meaning for the sample. One run might observe a white cell before a red cell; another run might observe a red cell first. The interpretation should come from the collection of measured events, such as proportions, counts, and feature distributions, not from the arbitrary order of rows.

In other words, if the same cell rows are listed in a different order, the blood sample is still the same sample. Hence, as in Zhang 2022, for a sample-level RBC prediction, row order should not decide the answer.

Scope limit This lesson does not teach instrument physics, gating, debris removal, or doublet handling. It only teaches the set abstraction needed for Set Transformer.

Three Behaviors Under Shuffling

A permutation is a reordering of the elements. When you shuffle the same cell rows, a good sample-level model should behave in a predictable way.

Permutation invariant

The output stays exactly the same when input rows are reordered.

f([a, b, c]) = f([c, a, b])

Permutation equivariant

The output rows reorder in the same way as the input rows.

g([a, b, c]) = [A, B, C]
g([c, a, b]) = [C, A, B]

Order-sensitive

The output changes just because the rows were listed in a different order.

r([a, b, c]) != r([c, a, b])

What Are Set Models?

Set models are models designed for permutation-invariant or permutation-equivariant behavior. Deep Sets is the baseline architecture to understand first: it encodes each element, pools across elements, and then decodes the pooled representation.

The Baseline: Deep Sets

Zaheer 2017 presents the standard pooling pattern for set-input models: encode each element, pool across elements, then decode the pooled representation.

prediction = decoder(pool(encoder(each event row)))

More precisely, if each cell starts with four raw features, the encoder phi usually maps each row into h hidden features before pooling: n x 4 -> n x h -> 1 x h. The visual below keeps the feature dimension as four for simplicity, so you can focus on the row dimension disappearing during pooling.

Pooling gives the right symmetry for sample-level prediction: after pooling, the final answer does not depend on row order. The limitation is that simple pooling can lose information about interactions between event rows. Set Transformer addresses that limitation by using attention, which is the topic of Lesson 2.

Final Check

Say this without looking

A sample-level model for RBC event rows should be permutation invariant because the same blood sample is represented by the same set of events even if the rows are listed in a different order.

An event-level encoder can be permutation equivariant because it keeps one representation per event, and those representations should reorder with the input rows.
