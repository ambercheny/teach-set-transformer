Lesson 1

Permutation Invariance For Flow Cytometry

Learn the one symmetry that makes set models useful for flow cytometry: reordering cell-event rows should not change a sample-level prediction.

The Win

By the end of this lesson, you should be able to classify a function as permutation invariant, permutation equivariant, or order-sensitive. You should also be able to explain why hematocrit prediction from many RBC event rows is a set-input problem.

Vocabulary checkpoint Keep the glossary open if the words set, event, sample, invariant, or equivariant feel unstable.

Set, Not Sequence

A sequence has meaningful order. A sentence changes when its words are shuffled. A time series changes when its time points are shuffled.

A set has no meaningful order. If the same cell-event rows are listed in a different order, the blood sample is still the same sample. For a sample-level RBC prediction, row order should not decide the answer.

What An Event Represents

A flow cytometry event is one detected particle measurement. In this teaching workspace, after preprocessing, one retained event is treated as one cell row.

For the Flow-RBC framing in Zhang 2022, the useful abstraction is simple: one sample contains many RBC-level rows, each row has measured features, and the target is a sample-level hematocrit value.

Scope limit This lesson does not teach instrument physics, gating, debris removal, or doublet handling. It only teaches the set abstraction needed for Set Transformer.

Three Behaviors Under Shuffling

Invariant

The output stays exactly the same when input rows are reordered.

f([a, b, c]) = f([c, a, b])

Equivariant

The output rows reorder in the same way as the input rows.

g([a, b, c]) = [A, B, C]
g([c, a, b]) = [C, A, B]

Order-sensitive

The output changes just because the rows were listed in a different order.

r([a, b, c]) != r([c, a, b])

The Baseline: Deep Sets

Lee 2019 presents the standard pooling pattern for set-input models: encode each element, pool across elements, then decode the pooled representation.

prediction = decoder(pool(encoder(each event row)))

This gives the right symmetry for sample-level prediction: after pooling, the final answer does not depend on row order. The limitation is that simple pooling can lose information about interactions between event rows. Set Transformer addresses that limitation by using attention, which is the topic of Lesson 2.

Final Check

Say this without looking

A sample-level model for RBC event rows should be permutation invariant because the same blood sample is represented by the same set of events even if the rows are listed in a different order.

An event-level encoder can be permutation equivariant because it keeps one representation per event, and those representations should reorder with the input rows.
