Lesson 2

How Set Transformer Builds An Invariant Predictor

Learn how attention blocks keep event-level representations equivariant, then use learned pooling to produce one invariant sample-level prediction.

The Win

By the end of this lesson, you should be able to trace a set of cell rows through Set Transformer and name where equivariance and invariance happen.

Reference checkpoint Keep the architecture reference and glossary open. This lesson uses the same block names: MAB, SAB, ISAB, and PMA.

From Deep Sets To Set Transformer

Deep Sets gives the right symmetry by encoding each row, pooling across rows, and decoding the pooled vector. Set Transformer keeps the same goal but changes the middle: it uses attention in addition to pooling so each row representation can depend on other rows before pooling.

Deep Sets: row encoder -> fixed pooling -> prediction

Set Transformer: attention encoder -> learned attention pooling -> prediction

This matters for flow cytometry because a sample is not just a bag of independent rows. The distribution of rows, rare subpopulations, and relationships among measurements can all matter for a sample-level target.

Attention

Attention is a weighted lookup. The model makes query, key, and value projections from the input. A query is compared with keys to produce attention weights, and those weights are used to combine values.

Query

The vector asking what information is relevant.

Key

The vector being compared against the query.

Value

The vector being averaged after attention weights are computed.

attention output = weighted sum of values

3. Use scores as weights

After softmax, each query row has weights over all value rows. A high weight means that value contributes more to that query's updated representation. In other words, a high weight means the two cells are highly correlated.

weights x V -> updated rows

4. Produce updated cell vectors

The result is one importance score vector per cell, representing the relevance between the cell and all other cells. That is why is permutation equivariant.

In Set Transformer, attention is used twice: first to let rows interact with other rows, and later to pool the whole set into one or more sample-level outputs.

Set Transformer Architecture

Imagine one blood sample represented by n cell rows. Each row is a cell, and each column is a measured cell feature.

The encoder output is not an importance score. It is a context-aware representation for each cell: each row has been updated using information from other rows. PMA then reduces the set of encoded rows into k learned summary tokens. The value of k is the number of summary tokens, not necessarily the number of outcome classes.

The Four Block Names

MAB

Multihead Attention Block. The reusable building block.

SAB

Set Attention Block. Self-attention over all rows: SAB(X) = MAB(X, X).

ISAB

Induced Set Attention Block. Uses m learned inducing points to reduce cost from O(n^2) to O(nm).

PMA

Pooling by Multihead Attention. Learned seed vectors query the encoded set to produce sample summaries.

Where Equivariance Becomes Invariance

SAB and ISAB preserve the row structure. If you shuffle input rows, the encoded output rows shuffle the same way. This is exactly what you want while the model is still representing each event separately.

PMA is different. PMA uses learned seed vectors to ask for a summary of the whole encoded set. That summary does not have one row per input event, so it can become a sample-level representation whose value should not depend on row order.

Practical rule Use equivariant blocks while you still need one representation per event. Use invariant pooling when you need one representation for the whole sample.

Why This Fits RBC Prediction

In the Flow-RBC framing from Zhang 2022, the input is a set of RBC-level measurements and the target is sample-level hematocrit. That makes row order irrelevant, but the distribution of cell measurements can still matter.

Set Transformer fits this shape: the encoder can learn event representations that account for other events, and PMA can aggregate them into one sample representation for the prediction head.

Zhang 2022 then asks a deeper training question: if we make permutation-invariant models deeper, how do we avoid gradient and normalization problems? That is where clean-path residual connections and set norm enter. Those details are beyond the core Set Transformer trace, but they matter when evaluating whether the architecture is appropriate for real-valued cytometry data.

Final Check

Say this without looking

Set Transformer first uses SAB or ISAB blocks to build permutation-equivariant event representations. Then PMA uses learned seed vectors to pool those event representations into a permutation-invariant sample representation. A final prediction head can map that sample representation to hematocrit or another sample-level target.
